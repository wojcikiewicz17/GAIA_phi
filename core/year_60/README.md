# core/year_60/ — Captura histórica do repositório

## Propósito
Subdiretório dedicado a scripts de captura de estado da base de código, útil para análise longitudinal.

## Estrutura de arquivos
- `capture_repository.py`: captura programática da estrutura do repositório.
- `capture_repository.c`: alternativa em C para captura/snapshot.

## Conceitos principais
- **Versionamento observável**: registrar a forma do repositório em pontos de controle.
- **Comparabilidade temporal**: facilitar diffs estruturais entre ciclos de evolução.
